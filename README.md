## О проекте

Voice-ka представляет собой десктопное приложение для голосового общения, разработанное в рамках учебного или дипломного проекта. Это мессенджер, который помимо стандартных возможностей голосового и видеосвязи предлагает пользователям уникальную функцию изменения голоса в реальном времени с помощью эквалайзеров, эффектов и микшера.

Проект рассчитан на широкую аудиторию, которая активно использует голосовые коммуникации в интернете. В первую очередь это геймеры, которым необходимо общение во время совместных игр, участники онлайн-конференций и рабочих встреч, стримеры и создатели контента, а также обычные пользователи, желающие разнообразить свое голосовое общение с друзьями и близкими.

**Актуальность проекта** обусловлена несколькими факторами. Современные популярные мессенджеры и платформы для голосового общения, такие как Discord, Skype или Zoom, предоставляют базовые возможности для связи, но почти не предлагают инструментов для обработки и изменения голоса в реальном времени. Пользователи вынуждены прибегать к сторонним программам-виртуальным микшерам, что создает дополнительные сложности в настройке и увеличивает нагрузку на систему. Voice-ka решает эту проблему, интегрируя все необходимые инструменты обработки звука непосредственно в приложение.

**Основные цели проекта:**
- Создание fully functional десктопного приложения с поддержкой голосовых и видеозвонков с минимальными задержками
- Разработка модуля обработки аудиопотока с возможностью применения различных эффектов и фильтров в реальном времени
- Обеспечение удобного и интуитивно понятного пользовательского интерфейса
- Реализация серверной части для управления пользователями и сопряжением вызовов
- Приобретение практических навыков работы с современными инструментами разработки

## Технологический стек

### Frontend
- **React 19** с TypeScript
- **Vite** для сборки
- **Tauri 2** для десктопной оболочки
- **React Router DOM** для навигации
- **CSS Modules** для стилизации
- **Vitest** + **React Testing Library** для unit-тестов

### Backend
- **Rust** с **Tokio** для асинхронности
- **WebSocket** (tokio-tungstenite) для real-time общения
- **PostgreSQL** с **SQLx** для работы с БД
- **Argon2** для хеширования паролей
- **Serde** для сериализации/десериализации

### Инфраструктура
- **Docker** для контейнеризации БД
- **Playwright** для E2E-тестирования
- **Git** для контроля версий

## Архитектура проекта

Проект построен по **клиент-серверной архитектуре** с использованием **WebSocket** для двусторонней связи.

### Структура каталогов
```
Voice-ka/
├── src-client/                 # Frontend-часть (React + Tauri)
│   ├── src/
│   │   ├── app/               # Корневые компоненты приложения
│   │   ├── entities/          # Сущности (Message, MessageInput)
│   │   ├── features/          # Бизнес-логика (auth, websocket, store)
│   │   ├── pages/             # Страницы приложения
│   │   ├── shared/            # Переиспользуемые компоненты
│   │   ├── widgets/           # Сложные виджеты (списки, поля)
│   │   └── test/              # Настройка тестов
│   └── src-tauri/             # Tauri-конфигурация
├── src-server/                # Backend-часть (Rust)
│   ├── src/
│   │   ├── db/               # Работа с базой данных
│   │   ├── handlers/         # Обработчики запросов (auth, room, message)
│   │   ├── ws/               # WebSocket-логика
│   │   └── lib.rs, main.rs
│   ├── migrations/           # SQL-миграции
│   └── tests/                # E2E-тесты (Playwright)
└── setup/                    # Скрипты для настройки окружения
```

### Коммуникация
1. **Клиент** подключается к WebSocket-серверу (порт 9001)
2. **Аутентификация** происходит через сообщения WebSocket с проверкой сессии
3. **Real-time сообщения** передаются через подписки на комнаты
4. **Состояние** хранится в локальном хранилище Tauri Store
5. **База данных** PostgreSQL используется для персистентного хранения

### Ключевые решения
- **WebSocket вместо HTTP/REST** — для минимальных задержек в голосовом общении
- **Tauri вместо Electron** — для меньшего размера приложения и лучшей производительности
- **SQLx с compile-time проверками** — для безопасности запросов к БД
- **Разделение на entities/features/widgets** — для соблюдения принципа единой ответственности

## Схема базы данных

```sql
-- Основные таблицы:
-- users - пользователи системы
-- sessions - активные сессии пользователей
-- guilds (каналы/серверы)
-- rooms (голосовые комнаты внутри каналов)
-- messages - текстовые сообщения
-- user_rooms - связь пользователей с комнатами (онлайн-статус)
-- user_guilds - связь пользователей с каналами (членство)
```

**Основные отношения:**
- Один **канал** содержит много **комнат**
- Одна **комната** может содержать много **сообщений**
- **Пользователи** могут быть участниками многих **каналов** и **комнат**

**Индексы:** созданы на часто используемых полях (user_id, room_id, guild_id, session_token) для оптимизации запросов.

## Быстрый старт

### Предварительные требования
- Docker Desktop
- Node.js 18+
- Rust (последняя стабильная версия)

### Первый запуск
1. Откройте Docker Desktop
2. Запустите `setup.bat` из папки `setup/` (это создаст и запустит контейнер PostgreSQL)
3. Установите зависимости клиента:
   ```bash
   cd src-client
   npm install
   ```
4. Убедитесь, что Rust установлен:
   ```bash
   cargo --version
   ```
5. **Выполните миграцию и заполните БД тестовыми данными:**
   ```bash
   cd src-server
   cargo run -- --migrate
   ```
6. Запустите клиентское приложение (в отдельном терминале):
   ```bash
   cd src-client
   npm run tauri dev
   ```
7. Запустите сервер (в другом терминале):
   ```bash
   cd src-server
   cargo run server
   ```

### Последующие запуски
1. Запустите контейнер `setup` в Docker Desktop
2. Выполните пункты 6-7 из предыдущего раздела

**Примечание:** Если нужно пересоздать БД с чистыми данными, выполните:
```bash
cd src-server
cargo run -- --migrate
```
Эта команда выполнит все миграции и заполнит БД тестовыми данными (пользователи, каналы, комнаты).

## Разработка

### Структура кода
- **Frontend**: React-компоненты используют хуки для состояния, CSS Modules для стилей
- **Backend**: Rust-модули разделены по ответственности (handlers, db, ws)
- **Типизация**: TypeScript strict mode, Rust с явными типами Result

### Code style
- **TypeScript**: ESLint не настроен, но используется strict mode
- **Rust**: форматирование через `cargo fmt`, линтинг через `cargo clippy`
- **Коммиты**: рекомендуется conventional commits

## Тестирование

### Unit-тесты (Frontend)
```bash
cd src-client
npm run test:unit          # Запуск тестов
```

### E2E-тесты (Backend)
```bash
cd src-server
npm test                   # Запуск Playwright-тестов
```

### Типы тестов
1. **Unit-тесты компонентов** (Vitest + React Testing Library)
   - Рендеринг компонентов
   - Взаимодействие с пользователем
   - Моки WebSocket-сервисов

2. **Интеграционные тесты WebSocket** (Playwright)
   - Аутентификация
   - Отправка/получение сообщений
   - Управление комнатами

## Команды приложения

| Команда | Описание | Где выполнять |
|---------|----------|---------------|
| `npm run tauri dev` | Запуск клиентского приложения | `src-client/` |
| `npm run test:unit` | Запуск unit-тестов | `src-client/` |
| `cargo run -- --migrate` | **Выполнение миграций БД и заполнение тестовыми данными** | `src-server/` |
| `cargo run server` | Запуск WebSocket-сервера | `src-server/` |
| `npm test` (server) | Запуск E2E-тестов | `src-server/` |
